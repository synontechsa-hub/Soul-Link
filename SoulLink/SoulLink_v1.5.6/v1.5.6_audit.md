## Stage 1 — Remove `_dev/` Runtime Dependencies
>
> **Run this first. Without it, production crashes immediately.**

Two files read from `_dev/` at runtime. Both must be fixed before the folder can be removed from deployment.

### Task 1.1 — Fix blueprint ingestion removed from startup

**File:** `backend/app/main.py` · Lines 62-66

The `SoulBlueprintService` reads all soul JSONs from `_dev/data/` on every server boot and upserts 135+ rows. It will crash with no `_dev/` folder. Seeders replace this entirely.

**Change:** Remove the blueprint ingestion call from the `lifespan` startup block. Replace with a DB healthcheck that warns if the `souls` table is empty.

```python
# REMOVE:
logger.info("Syncing Soul Blueprints from _dev/data...")
await blueprint_service.ingest_blueprints(session)

# REPLACE WITH:
result = await session.execute(text("SELECT COUNT(*) FROM souls"))
count = result.scalar()
if count == 0:
    logger.warning("⚠️ souls table is empty — run seed_all_v156.py before launch")
else:
    logger.info(f"✅ {count} souls loaded in DB")
```

---

### Task 1.2 — Fix `location_resolver.py` reading routines from disk per request

**File:** `backend/app/services/location_resolver.py` · Lines 88-109

`resolve_location()` opens `routines.json` from `_dev/` on every single-soul call. `resolve_bulk_locations()` has a `pass` stub — it skips Routine v2 entirely, silently falling back, meaning soul positions on the map are always wrong.

**Change — Part A:** Add a module-level startup cache for routines:

```python
# At module level:
_ROUTINES_CACHE: dict = {}

def initialize_routines(data: dict):
    """Called at startup by the system_config loader."""
    global _ROUTINES_CACHE
    _ROUTINES_CACHE = data
```

**Change — Part B:** Replace the `open()` call in `resolve_location()` with `_ROUTINES_CACHE.get(template_id, {})`.

**Change — Part C:** Implement the `pass` stub in `resolve_bulk_locations()` using the same cache instead of per-soul file reads.

**Change — Part D:** Call `initialize_routines()` at startup after loading `system_config` from DB (see Stage 4).

## Stage 2 — Fix Critical Bugs

### Task 2.1 — Fix world state cache dict mutation (user data bleed)

**File:** `backend/app/logic/time_manager.py` · Lines 118-150

The cached `soul_locations` dict is retrieved from cache and then mutated in-place with user-specific overrides. Any user who triggers a cache write after applying their overrides contaminates the shared cache entry that all other users read next.

**Change:** Make a private copy before applying user overrides:

```python
# Line 119 — change:
soul_locations = cached_data.copy()
# To (ensures no shared reference):
soul_locations = dict(cached_data)
```

## Stage 4 — DB Architecture: `system_config` Table

> **Goal:** All data in the database. No `_dev/` folder needed to run the app.

All 10 system JSONs move from disk into a `system_config` table. Services read from DB at startup and cache in-memory. Production Docker image ships `backend/` only.

### Task 4.1 — Create `SystemConfig` model

**File:** `backend/app/models/system_config.py` (**NEW**)

```python
class SystemConfig(SQLModel, table=True):
    __tablename__ = "system_config"
    key: str = Field(primary_key=True, max_length=100)
    data: Dict[str, Any] = Field(default_factory=dict, sa_type=JSON)
    version: str = Field(default="1.5.6", max_length=20)
    updated_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
```

---

### Task 4.2 — Refactor `WeatherService` to read from DB

**File:** `backend/app/services/weather_service.py`

Replace the `open()` at module load time with a DB fetch called once at startup. Keep file fallback for local dev without a seeded DB.

```python
async def initialize_from_db(session: AsyncSession):
    row = await session.get(SystemConfig, "weather")
    if row:
        _load_weather_cache(row.data)
    else:
        logger.warning("WeatherService: falling back to file (DB not seeded)")
        _load_weather_cache_from_file()
```

Call this in `main.py` lifespan startup, after DB is confirmed reachable.

## Production Deployment (itch.io)

Once all stages are complete, the app runs with zero `_dev/` dependency:

```
1. Set environment variables (.env):
   GROQ_API_KEY, SUPABASE_DB_URL, SUPABASE_URL, SUPABASE_ANON_KEY,
   ARCHITECT_UUID, ENVIRONMENT=production

2. Seed the production database (one-time):
   python _dev/scripts/seed_all_v156.py

3. Deploy backend/ only — _dev/ is NOT shipped:
   uvicorn backend.app.main:app --host 0.0.0.0 --port 8000
```

The `_dev/` folder remains a development tool only — never shipped.
